Цели проекта

  • Исследовать существующие решения для управления проектами
  • Разработать прототип коммуникационной платформы для совместной работы над проектами
    • Проектирование прототипа платформы.
    • Подготовка инфраструктуры для разработки.
    • Разработка серверной части приложения.
    • Разработка клиентской части приложения.
    • Тестирование реализованной функциональности.
    • Подготовка отчета о проделанной работе.

Основные задачи ПО для управления проектами

  • Планирование.
  • Управление данными и предоставление информации.
  • Управление коммуникациями команды проекта.

Задачи ПО для управления проектами

Планирование

  • Планирование различных событий, зависящих друг от друга.
  • Планирование расписания работы сотрудников и назначение ресурсов на конкретные задачи.
  • Расчет времени, необходимого на решение каждой из задач.
  • Сортировка задач в зависимости от сроков их завершения.
  • Управление нескольким проектами одновременно.

Задачи ПО для управления проектами

Управление данными и предоставление информации

  • Список задач для сотрудников и информация о распределении ресурсов.
  • Обзор информации о сроках выполнения задач.

Задачи ПО для управления проектами

Управление коммуникациями команды проекта

  • Обсуждение и согласование рабочих вопросов проекта.
  • Предоставление доступа к информации о ходе проекта в виде живой ленты событий.

Типы ПО для управления проектами

  • По размещению:
    • Desktop (Настольные).
    • Web-based (Веб-приложения).
  • По количеству пользователей:
    • Персональные.
    • Однопользовательские.
    • Многопользовательские.
